# PostDate Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add user-defined `postDate` to the OP vocabulary so pages can carry their publication date, and repoint `?when` filters to use it.

**Architecture:** Four layers touched bottom-up: (1) harmonizer extraction adds `postDate` selectors, (2) indexing pipeline stores `octo:postDate` as a Unix timestamp, (3) SPARQL queries fetch and filter on `postDate`, (4) blobject output includes the new field. New `?created` and `?indexed` API params give expert access to the existing timestamps.

**Tech Stack:** Vitest, JSDOM, SPARQL/RDF, SvelteKit

```javascript
export const recordPostDate = async (s, value) => {
  if (!value) {
    return
  }
  const timestamp = new Date(value).getTime()
  if (isNaN(timestamp)) {
    return
  }
  await query(`
    delete {
      <${s}> octo:postDate ?o .
    } where {
      <${s}> octo:postDate ?o .
    }
  `)
  return await insert(`
    <${s}> octo:postDate ${timestamp} .
  `)
}
```
