# Page Deletion System Design

**Date:** 2026-03-30
**Status:** Draft
**Scope:** `packages/core/` and `scripts/` only — no SvelteKit routes, no public API. Admin/dev feature.

---

## Problem

The OP triplestore accumulates records for pages that no longer exist. When a URL returns 404 or another inaccessible response, its data should be cleanable — but only data *generated from that URL during indexing*, not data about it generated by other URLs.

---

## Constraints

- No public-facing API routes. Deletion is triggered via a standalone script or programmatically through the core library.
- Deletion must be scoped precisely: only triples where the dead URL is the **source** of the data.
- Both soft-delete (mark as unavailable) and hard-delete (remove triples) are supported.
- Error codes that trigger deletion are configurable.

---

## Data Model

### Soft Delete

Soft-deleting a page adds three predicates to its existing record. No other data is modified.

```
<url> octo:unavailable "404" .
<url> octo:unavailableSince 1743000000000 .
<url> octo:failCount 3 .
```

- `octo:unavailable` — the error code string that triggered the mark (e.g. `"404"`, `"410"`, `"timeout"`)
- `octo:unavailableSince` — Unix timestamp of the first failure
- `octo:failCount` — increments on each subsequent failed review

**Recovery:** if a soft-deleted URL returns 2xx on a later review, the three predicates are cleared and the page is re-indexed normally.

### Hard Delete Scope

Hard delete removes only triples generated *from* the URL during its own indexing:

- All triples with `<url>` as subject (metadata, outgoing octothorpes, outgoing mentions, blank nodes attached via `<url> octo:octothorpes _:bn`)
- `<origin> octo:hasPart <url>` — the parent–child link written during indexing
- **Origin special case:** if `<url>` is itself the registered origin (has `rdf:type octo:Origin`), also delete `octo:verified`, `octo:hasMember` triples, and the Origin type triple

**Not deleted:**
- Inbound links from other pages: `<other> octo:octothorpes <url>`
- Blank nodes from other pages that reference `<url>` as a target
- `<other> <url> <timestamp>` usage timestamps written by other pages' indexing

**Origin rule:** an Origin record is only cleaned up when the dead URL *is* the origin itself. If a subpage 404s, the Origin is untouched regardless of how many subpages remain.

## Architecture

### `packages/core/deleter.js` — `createDeleter` factory

Same dep-injection pattern as `createIndexer`. No SvelteKit imports.

```javascript
createDeleter({ query, queryArray, queryBoolean, instance })
```

Returns:

| Method | Description |
|--------|-------------|
| `softDeletePage(url, errorCode)` | Upserts unavailable markers; increments `failCount` on repeat calls |
| `hardDeletePage(url)` | Removes all triples generated from `url`; applies Origin special case |
| `clearSoftDelete(url)` | Removes `octo:unavailable`, `octo:unavailableSince`, `octo:failCount` |
| `isSoftDeleted(url)` | Returns boolean |
| `getSoftDeletedPages()` | Returns `[{ url, errorCode, since, failCount }]` for all soft-deleted pages |
| `getAllSubjectUrls()` | Returns all `octo:Page` URIs (used to seed the review script) |
| `shouldHardDelete(record, config)` | Pure function — no SPARQL. Returns `true` if either threshold is met |

`shouldHardDelete(record, config)` accepts:
- `record: { since: number, failCount: number }`
- `config: { maxAgeDays: number, maxRetries: number }`

Returns `true` if `Date.now() - since > maxAgeDays * 86400000` **or** `failCount >= maxRetries`.

### `packages/core/indexer.js` — inline 404 detection

`createIndexer` gains an optional `deleter` dep. When provided, `handler()` checks the HTTP response status after the fetch (step 7) against a configurable `deadCodes` array. On a match:

1. Calls `deleter.softDeletePage(url, status)`
2. Throws an error to halt further processing

This means normal indexing requests automatically soft-mark dead pages without requiring the review script.

### `packages/core/index.js` — `createClient` integration

`createClient` wires `createDeleter` alongside `createIndexer` and exposes:

```javascript
deleteSource(url, { mode: 'soft' | 'hard', errorCode })
```

The deleter is also passed into the indexer so inline 404 detection works automatically when using `createClient`.

## Review Script (`scripts/review.js`)

Standalone Node.js script. Run via OS cron or manually. Uses `createClient` with config read from `.env` via `node --env-file=.env`.

### CLI Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--dry-run` | false | Report only — no mutations |
| `--mode soft\|hard\|auto` | `auto` | Deletion mode. `auto` = soft first, promote on thresholds |
| `--dead-codes 404,410` | `404,410` | Comma-separated HTTP status codes (and `timeout`) to treat as dead |
| `--max-age-days N` | `30` | Days before soft-delete promotes to hard (auto mode) |
| `--max-retries N` | `3` | Fail count before soft-delete promotes to hard (auto mode) |
| `--concurrency N` | `5` | Parallel URL checks |

### Flow

1. Fetch all `octo:Page` URLs via `deleter.getAllSubjectUrls()`
2. For each URL (batched per `--concurrency`):
   a. If already soft-deleted: run `shouldHardDelete()`. If threshold met, hard-delete and continue (no fetch needed)
   b. Fetch the URL with a reasonable timeout
   c. If status matches `deadCodes` (or fetch throws): call `softDeletePage()` or `hardDeletePage()` per `--mode`
   d. If status 2xx and URL was previously soft-deleted: call `clearSoftDelete()`, re-index
3. Print structured report

### Report Format

```
Review complete: 142 URLs checked
  Dead (newly flagged):    3  [404: example.com/old-post, ...]
  Hard-deleted:            1  [gone.example.com]
  Recovered:               0
  Errors (fetch failed):   2
  Skipped (cooldown):      0

Dry-run mode: no changes were written.
```

## Out of Scope

- Public-facing deletion API routes
- User-initiated deletion (no accounts/auth in OP Core)
- Deleting Terms or Origins independently (only as a side effect of hard-deleting a page that is itself an Origin)
- Cascading deletion of inbound links from other pages
