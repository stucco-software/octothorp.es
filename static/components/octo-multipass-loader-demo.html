<octo-multipass-loader> Component

Drag-and-drop MultiPass loader - Just like the Explore page, this component accepts MultiPass JSON files or GIF files with embedded MultiPass data.

Drop a file, and it automatically extracts the query, loads results, and displays them. Perfect for creating shareable feed widgets!

Features

  • ✅ Drag and drop - Drop JSON or GIF files directly
  • ✅ File browser - Click to browse for files
  • ✅ Direct GIF URLs - NEW! Provide a GIF URL for click-to-load mode
  • ✅ GIF extraction - Automatically extracts MultiPass from GIF comment blocks
  • ✅ Auto-load - Instantly fetches and displays results after upload
  • ✅ GIF preview - Shows the GIF image alongside results
  • ✅ Metadata display - Shows title, description, and author from MultiPass
  • ✅ Reset button - Easy to load a different file
  • ✅ Multiple render modes - List, cards, or compact

Example: Cards Layout

Use render="cards" for a grid layout:

<octo-multipass-loader render="cards"></octo-multipass-loader>

Example: Compact Layout

Use render="compact" for inline links:

<octo-multipass-loader render="compact"></octo-multipass-loader>

Example: Custom Placeholder

Customize the upload prompt:

<octo-multipass-loader 
  placeholder="Drop your curated feed here">
</octo-multipass-loader>

Example: Direct GIF URL (NEW!)

You can now provide a direct URL to a MultiPass GIF. The component will display the GIF as a clickable button - click it to load and execute the embedded query! 

<octo-multipass-loader 
  gif="https://example.com/path/to/multipass.gif">
</octo-multipass-loader>

Perfect for:

  • Embedding curated feeds directly in your pages
  • Creating interactive feed galleries
  • Sharing feeds via image hosting services
  • Building "click to load" feed widgets

Note: When the gif attribute is provided, the component skips the upload interface and displays the GIF directly. Users click the GIF to load results.

How to Use

1. Include the component

<script type="module" src="https://octothorp.es/components/octo-multipass-loader.js"></script>

2. Add to your page

<octo-multipass-loader></octo-multipass-loader>

3. Users can drag and drop

  • MultiPass JSON files (.json)
  • GIF files with embedded MultiPass (.gif)

4. Available Attributes

Attribute Type Description Default
render String Display mode: list, cards, compact list
placeholder String Custom upload prompt text "Drop MultiPass JSON or GIF here"
gif String (URL) Direct URL to MultiPass GIF - displays clickable GIF instead of upload interface ""

Workflow: Create → Share → Embed

Step 1: Create MultiPass on Explore Page

  1. Go to /explore
  2. Build your query (tags, filters, etc.)
  3. Add custom metadata (title, description, author)
  4. Download MultiPass JSON or encode into GIF

Step 2: Share the File

  • Host the JSON file on your site
  • Or encode into a GIF and share on social media
  • Or email the file to recipients

Step 3: Embed the Loader

<!-- On any website -->
<script type="module" src="https://octothorp.es/components/octo-multipass-loader.js"></script>

Use Cases

1. Shareable Curated Feeds

Create a curated feed on /explore, encode it into a GIF, and share it. Anyone can drop the GIF into this component to see your feed.

2. Personal Feed Widget

Add this component to your blog. Create MultiPass configurations for different topics, and let visitors drag-drop to explore them.

3. Feed Gallery

Create a page with multiple loaders for different topic feeds. Users can drag their saved MultiPass GIFs to any loader to compare queries.

4. Social Media Integration

Encode MultiPass into GIFs and share on social media. When someone saves the GIF, they can drop it into any loader component to see the full feed.

GIF Encoding

To create a MultiPass GIF:

  1. Go to /explore
  2. Build your query
  3. Click "Make MultiPass GIF"
  4. Upload a GIF image
  5. Click "Encode & Download"
  6. The MultiPass is embedded in the GIF's comment block

The GIF looks like a normal image but contains all the query data. This component automatically extracts and executes the embedded query!

Comparison: Loader vs Regular MultiPass Component

Feature <octo-multipass> <octo-multipass-loader>
Accepts JSON string ✅ Yes (via attribute) ✅ Yes (via file)
Accepts JavaScript object ✅ Yes ❌ No
Drag and drop ❌ No ✅ Yes
GIF extraction ❌ No ✅ Yes
File browser ❌ No ✅ Yes
Best for Programmatic use User interaction

When to use which:

  • Use <octo-multipass> when you have the MultiPass object in JavaScript
  • Use <octo-multipass-loader> when users will upload/drag-drop files
